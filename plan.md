🔒 Fitur "Sticky" yang Bikin User Gabisa Lepas dari VMatch
Prinsip Utama: Data Accumulation + Workflow Dependency + Network Effect
Semakin lama pakai → semakin banyak data tersimpan → semakin susah pindah ke cara manual/platform lain.

1. RAB Builder (Rencana Anggaran Biaya) — Admin & Vendor
Kenapa sticky: Admin bikin RAB langsung di platform, item per item (material, qty, harga satuan, markup). Setelah 10-20 proyek, semua template RAB dan harga material tersimpan. Pindah = bikin ulang dari nol.

Fitur:

Template RAB per jenis proyek (kitchen set, wardrobe, dll)
Library material + harga (auto-suggest dari proyek sebelumnya)
Kalkulasi otomatis (subtotal, markup %, pajak, grand total)
Export ke PDF branded (logo VMatch)
Perbandingan RAB vs aktual (cost tracking)
2. Material & Supplier Database — Admin
Kenapa sticky: Setelah 6 bulan, admin punya database ratusan material + supplier + harga terbaru. Ini jadi "otak" bisnis mereka.

Fitur:

Katalog material internal (nama, kode, supplier, harga, lead time, foto)
Supplier directory (kontak, spesialisasi, rating, riwayat order)
Price history (track harga material naik/turun)
Auto-suggest material saat bikin RAB
Reorder alert (material yang sering dipakai)
3. Daily Log / Laporan Harian — Vendor
Kenapa sticky: Vendor catat SEMUA aktivitas harian di sini. Setelah sebulan, ini jadi satu-satunya sumber kebenaran progress proyek.

Fitur:

Form harian: cuaca, jam kerja, aktivitas, kendala, foto
Auto-generate weekly report dari daily logs
Client bisa lihat summary mingguan (bukan detail harian)
Admin bisa lihat semua daily log
Histori lengkap per proyek (audit trail)
4. Kontrak Digital & E-Signature — Semua Role
Kenapa sticky: Semua kontrak tersimpan di platform. Kalau ada dispute, buktinya di sini. Gabisa pindah tanpa kehilangan legal trail.

Fitur:

Template kontrak (bisa di-customize admin)
Client tanda tangan digital (ketuk/draw signature)
Timestamp + IP log sebagai bukti
Kontrak otomatis ter-link ke proyek
Reminder jika kontrak belum ditandatangani
5. Warranty & After-Sales Tracking — Client
Kenapa sticky: Setelah proyek selesai, client TETAP buka platform untuk klaim garansi. Ini bikin mereka stay bahkan setelah proyek done.

Fitur:

Setiap proyek selesai → otomatis generate warranty card (misal: 1 tahun garansi)
Client bisa submit klaim garansi (foto kerusakan + deskripsi)
Admin/vendor review dan jadwalkan perbaikan
History klaim tersimpan
Reminder garansi mau habis → upsell maintenance package
6. Scheduling & Gantt Chart — Admin & Vendor
Kenapa sticky: Semua jadwal produksi, instalasi, dan koordinasi antar vendor ada di sini. Tanpa ini = chaos.

Fitur:

Timeline visual per proyek (Gantt-style)
Dependency antar task (misal: "Instalasi" gabisa mulai sebelum "Produksi" selesai)
Assign task ke vendor/tukang tertentu
Auto-notify jika ada delay
Calendar view semua proyek (admin bisa lihat kapasitas)
7. Client Selection Board / Approval System — Client
Kenapa sticky: Client pilih material, warna, handle, dll langsung di platform. Semua keputusan tercatat. Ini jadi "single source of truth".

Fitur:

Admin upload opsi material (3-5 pilihan per item)
Client pilih + approve di platform
Timestamp approval (bukti client sudah setuju)
Jika client minta revisi setelah approve → tercatat sebagai change order (bisa kena biaya tambahan)
Mengurangi "saya ga pernah setuju itu" drama
8. Change Order Management — Admin & Client
Kenapa sticky: Setiap perubahan setelah kontrak = change order. Ini protect bisnis dan bikin platform jadi legal backbone.

Fitur:

Client request perubahan (via form)
Admin review → kalkulasi biaya tambahan
Client approve change order + biaya tambahan
Auto-update RAB dan timeline
History semua change orders per proyek
9. Automated Notifications & Reminders — Semua Role
Kenapa sticky: Platform jadi "asisten" yang remind semua orang. Tanpa ini, semua harus manual via WA.

Fitur:

Auto-remind client: "Invoice jatuh tempo 3 hari lagi"
Auto-remind vendor: "Deadline proyek X tinggal 2 minggu"
Auto-notify admin: "Vendor belum update progress 3 hari"
Auto-notify client: "Ada progress baru di proyek kamu"
Weekly digest email (summary aktivitas minggu ini)
10. Analytics & Reporting Dashboard — Admin
Kenapa sticky: Setelah 6 bulan, admin punya data bisnis lengkap. Revenue, profit margin, vendor performance, client acquisition. Ini jadi "business intelligence" mereka.

Fitur:

Revenue per bulan/quarter
Profit margin per proyek (RAB vs aktual)
Vendor performance score (on-time delivery, rating)
Client acquisition funnel (request → deal conversion rate)
Average project duration
Top material/supplier yang paling sering dipakai
11. Referral & Loyalty Program — Client
Kenapa sticky: Client yang sudah selesai proyek bisa refer teman → dapat diskon untuk proyek berikutnya. Bikin mereka stay di ecosystem.

Fitur:

Unique referral link per client
Track siapa yang refer siapa
Reward: diskon X% untuk proyek berikutnya
Leaderboard referral (gamification)
12. Photo Documentation Timeline — Semua Role
Kenapa sticky: Semua foto progress dari hari 1 sampai selesai tersimpan rapi. Ini jadi portfolio vendor DAN kenangan client. Gabisa di-replicate di WhatsApp.

Fitur:

Auto-organize foto per tanggal per proyek
Before/after comparison slider
Client bisa share ke sosmed (watermark VMatch)
Vendor bisa pakai sebagai portfolio
Admin bisa pakai untuk marketing/testimonial
